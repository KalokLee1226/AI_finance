# 更新日志

## [2026-03-25] - v0.3

### 新增功能
1. 智能技术预警与邮件告警
	- 后端新增基于 RSI / MACD / BOLL 的智能预警模块，为用户自选品种生成提示级、预警级技术信号，并通过 `/api/alerts` 对外提供统一接口。
	- 支持通过 SMTP 在前端 Dashboard 一键“邮件发送本次预警”，仅在用户主动点击时触发邮件推送，避免骚扰。

2. 量化价格预测与多模型融合
	- 新增 `/api/predict-price/{symbol}` 接口，基于近端收益、波动率、RSI、MACD 等规则引擎给出未来 1–3 日方向与收益率区间。
	- 引入 ARIMA 计量模型，与规则视角做 60%/40% 权重融合，输出融合后的方向、区间和置信度，并在返回体中保留各单模型视图。
	- 前端 Dashboard 新增“短期量化预测”和“超短期波动评估（5–30 分钟）”卡片，配置信度进度条、中文解释，以及“技术规则 vs ARIMA”多模型对比小面板。

3. 多源新闻聚合与健康检查
	- 新闻模块接入 NewsAPI / GNews 免费 API，并扩展 HTML 源至 BBC、Reuters、CNBC、Yahoo Finance、CoinDesk 以及新浪财经、东方财富等站点，采用 `httpx + asyncio` 并发抓取 + 去重 + 按时间排序。
	- 当所有外部源暂不可用时自动返回一条“系统提示”型占位新闻，保证接口仍然可用而非前端空白。
	- 新增 `/api/news-debug` 自检接口与 `NEWS_VERIFY_SSL` 开关，便于在证书或内网代理环境下快速排查各新闻源连通性。

### 优化
1. 统一配置与安全
	- 后端启动时自动加载 `backend/.env`，集中管理 DeepSeek、AI_FINANCE_SECRET、NewsAPI、GNEWS、SMTP 等敏感配置，避免在代码中硬编码密钥。

2. 日志与稳定性
	- 降低 yfinance / investpy / akshare 以及 HTML 新闻抓取失败的日志级别，压制非致命的“上游数据源失败”刷屏，仅在真正不可恢复时输出错误日志。
	- 市场数据与新闻接口在部分数据源失败时自动尝试其他数据源或返回友好提示，提高整体鲁棒性和用户体验。
